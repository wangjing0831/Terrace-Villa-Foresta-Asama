## 実施内容

### 1. MySQL 5.7 互換性修正（前回セッションより）

**問題**: `ALTER TABLE plans ADD COLUMN IF NOT EXISTS` は MySQL 8.0+ の構文
→ RDS MySQL 5.7 で SQL エラーが発生し、新カラムが追加されなかった

**修正**: `INFORMATION_SCHEMA.COLUMNS` チェックに変更
- ファイル: `frontend/src/lib/db.ts`
- 対象カラム: `prestige_zh/ja/en`, `accommodation_images`, `conclusion_zh/ja/en`

---

### 2. プラン詳細画面のバグ修正（前回セッションより）

**問題1**: `force-dynamic` 未設定によるプリレンダリングキャッシュ
→ `X-Nextjs-Prerender: 1` / `Cache-Control: s-maxage=31536000` で古い HTML が返され続けていた

**修正1**: `export const dynamic = 'force-dynamic'` を追加
- ファイル: `frontend/src/app/plans/[id]/page.tsx`

**問題2**: `notFound` 状態がリセットされないバグ
→ `planId` が undefined の初回レンダリング時に `setNotFound(true)` が実行され、
　その後 `planId` が確定して API が成功しても「Plan Not Found」が表示されたまま

**修正2**: `load()` 開始時に `setNotFound(false)` を追加、`planId` ガード追加
- ファイル: `frontend/src/app/plans/[id]/page.tsx`

---

### 3. Admin 画像カテゴリ編集機能の追加

**問題**: 画像アップロード時のカテゴリが `uncategorized` に固定されており、
鉛筆アイコンの編集ダイアログでもカテゴリを変更できなかった
→ `/api/media/images?category=plan-golf` が常に空を返す原因

**修正**:
- 画像編集モーダルにカテゴリ選択ドロップダウンを追加
- 選択肢: `uncategorized` / `hotel` / `surroundings` / `plan-{id}`（動的生成）
- 保存時に `category` フィールドも API に送信するよう変更
- ファイル: `frontend/src/app/admin/page.tsx`
- コミット: `e6029f0`

---

### 4. ECS フロントエンドのハング対応

**症状**: `ERR_EMPTY_RESPONSE` — TCP 接続は確立するが HTTP レスポンスが返らない
**原因**: メモリ不足（512MB）による Next.js プロセスのハング

#### 4-1. ECS タスクメモリ増強
| サービス | 変更前 | 変更後 | タスク定義 |
|---------|--------|--------|-----------|
| Frontend | 256 CPU / 512MB | **512 CPU / 1024MB** | revision 6 |
| Backend  | 256 CPU / 512MB | **512 CPU / 1024MB** | revision 2 |

#### 4-2. ALB ヘルスチェックを HTTP に変更
TCP チェックではハングを検知できなかったため HTTP チェックに変更

| サービス | ヘルスチェックパス | 判定 |
|---------|-----------------|------|
| Frontend | `GET /api/plans` | 2成功→healthy、3失敗→unhealthy＋自動再起動 |
| Backend  | `GET /actuator/health` | 同上 |

---

### 5. `/plans/golf` 静的ページの問題調査

**発覚した問題**:
`frontend/src/app/plans/golf/page.tsx` という静的ルートが存在し、
`/plans/golf` へのアクセスは動的ルート `plans/[id]/page.tsx` ではなく
この静的ページが優先的に表示されていた

**静的ページの仕様**:
- ゴルフコース情報（5コース）、日程（7日分）、予算 — すべてコード内にハードコード
- 画像のみ `/api/media/images?category=plan-golf` から取得（DB参照）
- Admin で設定した内容は一切反映されない

**画像が表示されない理由**:
`media` テーブルに `category=plan-golf` の画像が1枚も存在しないため

**対応方針（明日実施予定）**:
1. `plans/golf/page.tsx` を削除（静的ページ廃止）
2. 静的ページ内のコンテンツ（ゴルフコース5件・日程7日・予算・翻訳）を DB に登録
3. デプロイ・動作確認
