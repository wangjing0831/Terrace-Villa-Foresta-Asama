# Work Log — 2026-03-12

## 対応内容

### 1. Admin ログイン後に画面遷移しない問題を修正

**原因:**
- `proxy.ts` が `middleware.ts` という正しいファイル名でなかったため Next.js がミドルウェアとして認識していなかった
- Cookie の `secure: true` が HTTP 接続（ALB 経由）で機能せず、ブラウザが Cookie を保存しなかった

**修正:**
- `frontend/src/proxy.ts` → `frontend/src/middleware.ts` にリネーム
- エクスポート関数名を `proxy` → `middleware` に変更
- `secure` フラグを `X-Forwarded-Proto` ヘッダーで判定するよう変更（HTTP では `false`、HTTPS では `true`）

**コミット:** `f1ec577`, `9627ebb`

---

### 2. `/api/media/upload` 500 エラーを修正 (S3 CredentialsProviderError)

**原因:**
- ECS タスク定義に `taskRoleArn` が未設定
- `executionRoleArn`（イメージ取得・ログ用）と `taskRoleArn`（コンテナ内 AWS API 呼び出し用）は別物
- タスクロールなしでは S3 への PutObject ができない

**修正:**
- IAM ロール `foresta-asama-ecs-task-role` を新規作成
  - 信頼ポリシー: `ecs-tasks.amazonaws.com`
  - インラインポリシー `S3MediaAccess`: `s3:PutObject`, `s3:GetObject`, `s3:DeleteObject`, `s3:ListBucket` on `terrace-villa-foresta-asama-assets`
- ECS タスク定義リビジョン 4 に `taskRoleArn` を追加
- ECS サービスを revision 4 に更新・デプロイ

---

### 3. `/api/auth/login` 404 エラーを修正

**原因 (前セッションから継続):**
- ALB リスナールールで `/api/*` が Java バックエンドサービスにルーティングされていた
- 全 API ルートは Next.js フロントエンドにあるのに、バックエンドに転送されていた
- ECS タスク定義に DB 環境変数が未設定
- ECS フロントエンドの SG が RDS の SG インバウンドルールに含まれていなかった

**修正:**
- ALB リスナールール (priority 10: `/api/*` → backend) を削除
- デフォルトルール（frontend target group）が全 `/api/*` を処理するよう変更
- ECS タスク定義リビジョン 3 に DB/S3 環境変数を追加:
  - `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASS`, `DB_NAME`, `DB_SSL`
  - `S3_BUCKET`, `AWS_REGION`
- RDS SG (`sg-009e3b3fa9d7bd092`) に ECS frontend SG (`sg-0a77a3303b6835b05`) からの port 3306 インバウンドを追加

---

### 4. 画像プレビュー修正 (S3 公開アクセス設定)

**原因:**
- S3 バケット `terrace-villa-foresta-asama-assets` が完全プライベート設定
- アップロードされた画像 URL (`https://...s3.ap-northeast-1.amazonaws.com/uploads/...`) が 403 Access Denied

**修正:**
- PublicAccessBlock: `BlockPublicPolicy=false`, `RestrictPublicBuckets=false` に変更（ACL ブロックは維持）
- バケットポリシーを追加: `uploads/*` プレフィックスへの `s3:GetObject` を全公開
