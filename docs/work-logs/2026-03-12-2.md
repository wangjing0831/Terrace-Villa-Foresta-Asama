### 2. Admin ログイン不具合の修正

**原因**: `users` テーブルに管理者ユーザーが存在しない（DB 未初期化）

**対応**:
- `ADMIN_USER` / `ADMIN_PASS` 環境変数フォールバック認証を追加
  （DB 不要でログイン可能）
- DB エラー時は 500 ではなく 401 を返すよう改善
- ECS タスク定義 `:5` に環境変数を追加（`aws ecs register-task-definition`）
  - `ADMIN_USER=admin`
  - `ADMIN_PASS=ForestAsama2025!`
- ファイル: `frontend/src/app/api/auth/login/route.ts`

---

### 3. Plan 詳細画面のギャラリー画像が表示されない問題の修正

**原因1**: ギャラリー取得ロジックが `/api/media/images` との URL 突き合わせを必要としていた
- DB エラー時に `/api/media/images` が空配列を返すと全画像が除外された

**修正1**: URL 突き合わせを廃止し、layout URL を直接使用
- ファイル: `frontend/src/app/plans/[id]/page.tsx`

**原因2**: API レスポンスに `Cache-Control` ヘッダーがなく、ブラウザが 304 (Not Modified) を返してキャッシュを使用
- 古いキャッシュ（`plan.golf.gallery` なし）が使用されていた

**修正2**: 動的データ API に `Cache-Control: no-store` を追加
- `frontend/src/app/api/layouts/route.ts`
- `frontend/src/app/api/media/images/route.ts`
- `frontend/src/app/api/plans/[id]/route.ts`

---

### 4. DB スキーマ確認・根本原因特定

- `foresta-asama-schema-setup` Lambda を再実行 → `"Schema foresta_asama and user foresta_user created."` を確認
- セキュリティグループ確認: ECS (`sg-0a77a3303b6835b05`) → RDS (`sg-009e3b3fa9d7bd092`) の 3306 接続が許可済みであることを確認
- **根本原因判明**: `CREATE TABLE` 文がコードベースに存在しなかった
  → Lambda はDB・ユーザーのみ作成、テーブル未作成のため全 DB 操作がサイレント失敗

---

### 5. DB テーブル自動作成の実装

**対応**:
- `frontend/src/lib/db.ts` に `runMigration()` 関数を追加
  → `media`, `page_layouts`, `plans`, `users` の 4 テーブルを `CREATE TABLE IF NOT EXISTS` で自動作成
- `frontend/src/instrumentation.ts` を新規作成
  → Next.js サーバー起動時（リクエスト処理前）に `runMigration()` を実行
- デプロイ後の CloudWatch ログで `[db] migration complete` を確認 → テーブル作成成功

---
